# JivniCare Component Brand Mapping

Version: 1.0

This document defines the official branding asset for every page, layout, component, state, and workflow.

This document overrides personal design decisions.

If this document conflicts with any design assumption, this document wins.

---

# Global Rules

Every component must use the approved asset defined below.

Never replace an asset because it "looks better."

Never use branding as decoration.

Maximum one branding element per logical section.

Maximum one primary logo per viewport.

# Strict Prohibitions

Never place branding inside:

- Feature Cards
- Statistics Cards
- Pricing Cards
- FAQ Cards
- Doctor Cards
- Hospital Cards
- Appointment Cards
- Dashboard Widgets
- Analytics Cards
- Search Results
- Tables
- Charts

Branding belongs to navigation, identity, authentication, loading, documents, and official communication.

It must never be used as decorative artwork.

# Final Rule

If a component is not listed in this document,

STOP.

Update this document first.
